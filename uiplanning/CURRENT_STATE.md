## Architecture Facts

| Fact | Detail |
|------|--------|
| Language | Pure Java 17, zero Kotlin |
| Activities | Exactly 2: `MainActivity`, `DetailActivity` |
| Fragments | Zero |
| Navigation | Manual `startActivity()` / `finish()` |
| List widget | `ListView` + `ArrayAdapter` (NOT RecyclerView) |
| Persistence | Raw `SQLiteDatabase` for pack data; zero session persistence |
| Session store | In-memory `LinkedHashMap<String, SessionMemory>`, LRU max 10 |
| Threading | `ExecutorService` (single thread) + `runOnUiThread()` |
| Theme | `Theme.AppCompat.Light.DarkActionBar`, fully custom drawables |
| Material Design | Not a dependency — no `com.google.android.material` |
| Min SDK | 26 · Target SDK 34 · Compile SDK 34 |

## Category Mapping (in SearchResultAdapter)

Only 4 categories get visual treatment:
- `"water"` → `senku_badge_water` `#6B88A6`
- `"fire"` → `senku_badge_fire` `#B56B3E`
- `"medical"` → `senku_badge_medicine` `#8E5A5F`
- `"shelter"` / `"building"` → `senku_badge_shelter` `#5F7B52`
- everything else → `senku_accent_olive` `#6B7A4A`

The briefings propose adding Food, Tools, Communications, Community as named
categories. The pack data includes broader categories (survival, chemistry,
crafts, agriculture, resource-management, defense) but the adapter only maps 4.

## Deterministic Rules

96 rules in `deterministic_special_case_registry.py`. The Android app's
`DeterministicAnswerRouter.java` has 9 hardcoded rules. The pack's
`deterministic_rules` SQLite table holds all 96 but the app doesn't use the
table — it uses its own hardcoded Java predicates.

## Harness Contract IDs

The following IDs are used by PowerShell automation harnesses that tap the UI
via `uiautomator`. These MUST remain stable or the harnesses break:

- `search_input`, `search_button` — prompt injection
- `results_list` — result verification
- `detail_title`, `detail_body` — answer verification
- `detail_followup_input`, `detail_followup_send` — follow-up automation
- `detail_sources_container` — source citation checks

Any redesign must either preserve these IDs or update all harness scripts.
